# CarRacing-v3: Решение с помощью Rainbow DQN

Проект посвящен обучению агента для среды `CarRacing-v3` из библиотеки Gymnasium. В качестве алгоритма обучения используется **Rainbow DQN**, который объединяет в себе несколько улучшений классического алгоритма Deep Q-Network (DQN). 

В результате обучения агент способен уверенно проходить трассу, достигая **среднего скора около 850 баллов**. Основная логика работы, обучения и тестирования находится в файле `car-racing-c51.ipynb`.

## Описание архитектуры агента

Агент построен на основе архитектуры **Rainbow**, которая включает в себя следующие компоненты (все они реализованы в проекте):

1. **Double DQN**: Использование двух сетей (локальной и целевой) для уменьшения переоценки Q-значений. Локальная сеть выбирает лучшее действие, а целевая сеть оценивает его.
2. **Dueling DQN**: Разделение нейросети на два потока — оценка ценности состояния (Value) и оценка преимущества каждого действия (Advantage). Это позволяет агенту быстрее понимать, какие состояния хороши сами по себе, независимо от действий.
3. **Categorical DQN (C51)**: Оценка не просто ожидаемого значения (скаляра), а полноценного распределения вероятностей будущих наград на заданном носителе (от `V_MIN` до `V_MAX` разбитом на `N_ATOMS = 51`).
4. **Noisy Nets (NoisyLinear)**: Использование зашумленных линейных слоев вместо классической $\epsilon$-greedy стратегии. Шум добавляется непосредственно в веса сети, что стимулирует направленное, зависящее от состояния исследование (exploration).
5. **Prioritized Experience Replay (PER)**: Использование буфера памяти на основе `SumTree`, который отдает приоритет тем переходам (transitions), на которых нейросеть сильнее всего "ошибается" (где ошибка TD больше).
6. **Multi-step Learning ($N$-step)**: Вычисление целевого значения с учетом накопленной награды за $N$ шагов (по умолчанию `n_step=3`). Это позволяет быстрее распространять информацию о наградах по траектории.

## Предобработка среды

Среда `CarRacing-v3` выдает наблюдения в виде RGB-изображений 96x96. Для ускорения обучения и улучшения сходимости используется пайплайн предобработки (через wrappers Gymnasium):

* **`SkipFrame(env, skip=4)`**: Агент выбирает действие каждый 4-й кадр, а выбранное действие повторяется в пропущенных кадрах. Это ускоряет симуляцию и делает среду менее высокочастотной.
* **`CropObservation(env)`**: Обрезает нижнюю часть экрана (12 пикселей), убирая счетчик очков, скорость и руль, чтобы агент не пытался "запомнить" их вместо анализа трассы.
* **`GrayscaleObservation(env)`**: Переводит изображение в оттенки серого (убирает лишние цветовые каналы).
* **`ResizeObservation(env, (84,84))`**: Изменяет размер изображения к стандартному формату 84x84 пикселя.
* **`FrameStackObservation(env, 4)`**: Объединяет 4 последних кадра в один тензор (состояние). Это дает агенту понимание динамики: скорости и направления движения машины.

## Установка и зависимости

Для работы проекта необходимо установить библиотеку `gymnasium` с поддержкой физического движка `box2d`, а также PyTorch и другие вспомогательные пакеты.

Выполните следующую команду перед запуском кода:

```bash
pip install swig "gymnasium[box2d]" torch numpy matplotlib imageio
```

## Запуск проекта

Вся логика инкапсулирована в Jupyter Notebook `car-racing-c51.ipynb`.

### 1. Обучение (Train)
В блоке `# Train` создается среда и инициализируется агент:
* Создаются локальная и целевая сети с архитектурой `DuelingNoisyQNetwork` (включая конволюционные слои для обработки изображений).
* Агент обучается в течение 800+ эпизодов (с сохранением чекпоинтов каждые 100 эпизодов).
* После завершения обучения строятся графики метрик и веса сети сохраняются (например, `best_model.pth`).

### 2. Тестирование (Test)
В блоке `# Test` можно проверить обученную модель:
* Загружаются предварительно сохраненные веса с помощью `torch.load()`.
* Агент переводится в режим оценки (`eval()`), чтобы отключить фактор случайности Exploration-шума.
* Прогоняется `num_episodes` эпизодов, результаты выводятся в консоль. Средней наградой является ~850.
